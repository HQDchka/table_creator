## Описание проекта

**Генератор расписания** — это веб-приложение для автоматического составления расписания занятий в учебных заведениях. Приложение позволяет управлять группами, предметами, преподавателями и автоматически распределять занятия с учётом ограничений.

### Ключевые возможности

- ✅ Создание и управление несколькими учебными группами
- ✅ Добавление предметов с указанием преподавателя и количества часов
- ✅ Настройка максимального количества пар в день для каждой группы
- ✅ Указание доступности преподавателей по дням и парам
- ✅ Автоматическая генерация расписания с учётом всех ограничений
- ✅ Drag & Drop для ручного редактирования расписания
- ✅ Визуальное выделение конфликтов преподавателей
- ✅ Сохранение/загрузка конфигурации в localStorage
- ✅ Экспорт расписания в Excel (XLSX)

## Инструкция пользователя

### Начало работы

1. Откройте файл `index.html` в современном веб-браузере (Chrome, Firefox, Edge, Safari)
2. Интерфейс разделён на несколько основных блоков:
   - Панель управления группами
   - Настройка предметов
   - Доступность преподавателей
   - Область генерации и экспорта

### Управление группами

#### Добавление/удаление групп

1. В поле **"Количество групп"** укажите нужное число (от 1 до 5)
2. Нажмите кнопку **"Применить группы"**

#### Настройка параметров группы

Для каждой группы доступны следующие параметры:

- **Название группы** — кликните по полю с названием для редактирования
- **Макс пар/день** — максимальное количество занятий в один день
- **Макс одинаковых/день** — сколько раз один предмет может встречаться в день

> ⚠️ **Рекомендация:** Для стандартной нагрузки устанавливайте 3-4 пары в день, максимум одинаковых — 2.

### Настройка предметов

Для каждой группы можно добавить предметы:

1. Нажмите **"+ Добавить предмет"** в карточке группы
2. Заполните поля:
   - **Название предмета** (например, "Математика")
   - **Преподаватель** (ФИО)
   - **Пар в неделю** (от 1 до 20)
3. Для удаления предмета нажмите **✖**

> 💡 **Совет:** Если преподаватель ведёт один предмет в нескольких группах, укажите одинаковое ФИО — система будет учитывать его занятость.

### Доступность преподавателей

#### Автоматическое обновление

1. После добавления предметов нажмите **"🔄 Обновить список преподавателей"**
2. Для каждого преподавателя отобразятся все дни недели

#### Настройка доступности

Укажите номера пар, когда преподаватель **свободен**:

- **Формат ввода:**
  - Отдельные номера: `1,3,5`
  - Диапазоны: `2-7`
  - Смешанный: `1,3-5,7`

- **Нумерация пар:** от 1 до 7

> 💡 **Важно:** Если поле оставить пустым, преподаватель считается доступным во все пары.

### Генерация расписания

1. Убедитесь, что все группы имеют хотя бы по одному предмету
2. Настройте рабочие дни недели (галочки вверху страницы)
3. Нажмите **"🎲 Сгенерировать расписание"**

#### Алгоритм генерации

Система пытается составить расписание с учётом:
- Количества часов по каждому предмету
- Занятости преподавателей
- Максимальной нагрузки на группу в день
- Доступности преподавателей

Если расписание не удаётся составить с первой попытки, алгоритм повторяет попытку до 800 раз.

### Редактирование расписания

#### Drag & Drop

- **Перетаскивание уроков:** зажмите левую кнопку мыши на уроке и перетащите в другую ячейку
- **Автосохранение:** после перемещения расписание обновляется автоматически

#### Визуальные индикаторы

- 🔵 **Синие ячейки** — обычные занятия
- 🔴 **Красные ячейки с ⚠️** — конфликт (один преподаватель ведёт два занятия одновременно)
- 📊 **Статистика** — количество пар по дням и число конфликтов

### Сохранение и загрузка

#### Сохранение конфигурации

Нажмите **"💾 Сохранить"** — все данные (группы, предметы, доступность преподавателей) сохранятся в браузере.

#### Загрузка конфигурации

Нажмите **"📂 Загрузить"** — восстановится последняя сохранённая конфигурация.

> ℹ️ **Примечание:** Данные хранятся в localStorage вашего браузера и не теряются после перезагрузки страницы.

### Экспорт в Excel

1. Сгенерируйте расписание
2. Нажмите **"📊 Экспортировать в Excel"**
3. Файл автоматически сохранится с названием в формате:  
   `Расписание_Названия_групп_дата.xlsx`

#### Особенности экспорта

- Каждая группа сохраняется на отдельном листе
- Конфликтующие занятия помечаются как **"⚠️ КОНФЛИКТ! Название предмета"**
- Добавляется строка-предупреждение при наличии конфликтов

### Модели данных

#### Группа (Group)

```javascript
{
  name: string,              // Название группы
  maxPairsPerDay: number,    // Максимум пар в день (1-7)
  maxSamePerDay: number,     // Максимум одинаковых предметов в день
  subjects: [                // Список предметов
    {
      name: string,          // Название предмета
      teacher: string,       // ФИО преподавателя
      hoursPerWeek: number   // Количество часов в неделю
    }
  ]
}
```

#### Расписание (Schedule)

```javascript
// schedule[groupIndex][dayIndex][pairIndex] = Lesson | null
Lesson: {
  name: string,
  teacher: string
}
```

#### Доступность преподавателя

```javascript
teacherAvailability: {
  "Иванов А.А.": {
    0: [0,1,2,3,4,5,6],  // Понедельник, доступные пары
    1: [0,1,2,3,4,5,6],  // Вторник
    // ... дни 2-5
  }
}
```

### Алгоритмы

#### 1. Генерация расписания

```javascript
Алгоритм:
1. Создать массив занятий для каждой группы (hoursPerWeek копий предмета)
2. Перемешать занятия случайным образом
3. Пока есть нераспределённые занятия:
   a. Перебрать группы в случайном порядке
   b. Для каждого занятия найти все возможные слоты с учётом:
      - Лимита пар в день
      - Лимита одинаковых предметов
      - Занятости преподавателя
      - Доступности преподавателя
   c. Выбрать первый подходящий слот
4. Если все занятия распределены → успех
5. Если нет → повторить с шага 2 (макс. 800 попыток)
```

**Сложность:** O(800 * N * M * K), где:
- N — количество групп
- M — количество предметов
- K — количество дней

#### 2. Поиск конфликтов преподавателей

```javascript
1. Создать Map: key = "teacher|day|pair" → список позиций
2. Для каждой пары "преподаватель-время":
   Если количество позиций > 1 → все позиции конфликтны
3. Вернуть Set конфликтных позиций
```

**Сложность:** O(G * D * P), где:
- G — количество групп
- D — количество дней
- P — количество пар

## Возможные проблемы и решения

### Проблема: Не удаётся сгенерировать расписание

**Причины:**
- Слишком много часов предметов при малом количестве дней
- Противоречивые ограничения (например, преподаватель недоступен, но часы есть)
- Максимальное количество пар в день слишком маленькое

**Решения:**
1. Увеличьте количество рабочих дней
2. Уменьшите часы некоторых предметов
3. Расширьте доступность преподавателей
4. Увеличьте лимит "Макс пар/день" для групп

### Проблема: Не работают конфликты преподавателей

**Причины:**
- У преподавателей разные написания ФИО (например, "Иванов А.А." и "Иванов А А")
- Пробелы или регистр символов

**Решение:** Используйте точное совпадение ФИО во всех предметах.

### Проблема: Не сохраняется конфигурация

**Причины:**
- Браузер в режиме "Инкогнито" или "Приватный"
- Отключён localStorage
- Превышен лимит хранилища

**Решение:** Проверьте настройки браузера или используйте обычный режим.

### Проблема: Не экспортируется Excel

**Причины:**
- Библиотека SheetJS не загрузилась
- Блокировка скриптов расширениями браузера

**Решение:** Проверьте консоль браузера (F12) на наличие ошибок.
